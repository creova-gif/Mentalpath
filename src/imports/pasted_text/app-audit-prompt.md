AI-Powered Pre-Launch App Audit Prompt
Role: Act as a senior product auditor with expertise in UX/UI, security, performance optimization, and market positioning.

Context: I'm about to launch [APP NAME] - a [brief 1-sentence description of what your app does] targeting [primary user demographic].

App Details:

Platform: [iOS/Android/Web/Cross-platform]

Key features: [List 3-5 main features]

Tech stack: [If known - e.g., React Native, Flutter, Native]

Development stage: [Beta/Ready to launch/Last mile polish]

AUDIT REQUIREMENTS:
Please conduct a comprehensive pre-launch audit covering these 7 critical areas:

1. SECURITY & COMPLIANCE
Identify potential vulnerabilities in authentication, data storage, and API communications

Check GDPR/CCPA compliance for user data handling

Audit permission requests (are they minimal and justified?)

Review payment/transaction security if applicable

2. USER EXPERIENCE (UX)
Analyze first-time user flow and onboarding friction points

Identify any confusing navigation patterns

Check accessibility standards (WCAG compliance)

Evaluate error handling and recovery paths

3. USER INTERFACE (UI)
Review visual consistency (colors, typography, spacing)

Check for responsive design issues across different devices

Identify any UI elements that might frustrate users

Verify micro-interactions and feedback mechanisms

4. PERFORMANCE & TECHNICAL
Identify potential bottlenecks in app loading time

Check memory usage and battery consumption patterns

Review offline functionality and sync mechanisms

Analyze API response times and error handling

5. ONBOARDING & RETENTION
Evaluate the value proposition clarity

Check for unnecessary barriers in sign-up process

Identify features that could improve user retention

Review push notification strategy (timing, relevance, frequency)

6. APP STORE OPTIMIZATION (ASO)
Review app title, subtitle, and keywords

Analyze screenshots and preview video effectiveness

Check category selection and app store description

Identify potential user objections to address in listing

7. COMPETITIVE POSITIONING
Compare core features against top 3 competitors

Identify unique selling propositions that aren't highlighted enough

Spot missing "table stakes" features users will expect

Suggest positioning improvements

DELIVERABLE FORMAT:
Provide your audit results as:

Critical Issues (must fix before launch)

Important Improvements (should address soon after launch)

Nice-to-Haves (consider for roadmap)

Quick Wins (easy changes with high impact)

For each finding, include:

The specific issue

Why it matters

Concrete recommendation for fixing

ADDITIONAL CONTEXT:
[Add any specific concerns, target metrics, or areas you're unsure about]

Please be brutally honest - I want to catch problems now rather than after launch!

This prompt is designed to give you actionable, comprehensive feedback that covers all the bases most startups miss in their rush to launch. The "brutal honesty" instruction helps override the AI's tendency to be overly positive.
